# 甄味 (FlavorGuide) 后端接口与数据库设计清单

> 本文档根据项目前端功能梳理，涵盖所有需要后端支撑的接口及对应的数据库表结构设计。

---

## 一、接口清单总览

共 **5 大模块、17 个接口**。

| 模块 | 接口数量 | 说明 |
|:---|:---:|:---|
| 分类模块 | 1 个 | 获取父分类+子分类树 |
| 菜谱模块 | 5 个 | 列表、详情、排行榜、推荐、搜索 |
| 评论模块 | 2 个 | 获取评论、发表评论 |
| 用户交互模块 | 5 个 | 点赞、收藏、查询我的收藏/评论/点赞 |
| 用户认证模块 | 4 个 | 登录、注册、获取/更新用户信息 |

---

## 二、详细接口定义

### 2.1 分类模块

#### 1. 获取分类列表
```
GET /api/categories
```
**说明：** 返回所有父分类及其下的子分类，用于首页快捷入口和分类页导航。

**响应示例：**
```json
{
  "code": 200,
  "data": [
    {
      "id": "p1",
      "name": "热门分类",
      "subCategories": [
        { "id": "c1", "parentId": "p1", "name": "家常菜", "image": "..." },
        { "id": "c2", "parentId": "p1", "name": "下饭菜", "image": "..." }
      ]
    }
  ]
}
```

---

### 2.2 菜谱模块

#### 2. 获取菜谱列表
```
GET /api/recipes
```
**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |
| categoryId | string | 否 | 子分类 ID，筛选该分类下的菜谱 |
| keyword | string | 否 | 搜索关键词（匹配标题、描述）|

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "list": [ /* Recipe 对象数组 */ ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

#### 3. 获取菜谱详情
```
GET /api/recipes/:id
```
**说明：** 返回完整菜谱信息，包含食材列表、制作步骤。

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "id": "r1",
    "title": "秘制红烧肉",
    "description": "...",
    "image": "...",
    "categoryId": "c1",
    "likes": 1254,
    "difficulty": "中等",
    "time": "60分钟",
    "servings": 4,
    "ingredients": [
      { "name": "五花肉", "amount": "500g" }
    ],
    "steps": [
      { "id": 1, "description": "...", "image": "...", "ingredientsUsed": ["五花肉"] }
    ]
  }
}
```

#### 4. 获取人气排行榜
```
GET /api/recipes/top-ranked?limit=3
```
**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| limit | int | 否 | 返回条数，默认 3 |

**说明：** 按点赞数降序排列，用于首页"人气排行榜"。

#### 5. 获取推荐菜谱
```
GET /api/recipes/recommended?page=1&pageSize=10
```
**说明：** 返回推荐菜谱列表（排除排行榜已展示的内容），用于首页"为你推荐"/"猜你喜欢"。

#### 6. 搜索菜谱
```
GET /api/recipes/search?q=xxx&page=1&pageSize=10
```
**查询参数：**
| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| q | string | **是** | 搜索关键词 |
| page | int | 否 | 页码 |
| pageSize | int | 否 | 每页条数 |

**说明：** 全局搜索，匹配标题和描述。注：也可合并到 #2 列表接口通过 keyword 参数实现。

---

### 2.3 评论模块

#### 7. 获取菜谱评论列表
```
GET /api/recipes/:id/comments?page=1&pageSize=20
```
**说明：** 返回指定菜谱的评论列表，按时间倒序。

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "list": [
      {
        "id": "cmt1",
        "recipeId": "r1",
        "userId": "u1",
        "username": "美食家",
        "avatar": "...",
        "text": "这红烧肉绝了！",
        "images": ["...", "..."],
        "date": "2026-04-18"
      }
    ],
    "total": 15
  }
}
```

#### 8. 发表评论
```
POST /api/recipes/:id/comments
```
**请求头：** `Authorization: Bearer <token>`

**请求体：**
```json
{
  "text": "评价内容",
  "images": ["url1", "url2"]
}
```

**说明：** 需登录，文字和图片至少填一项。

---

### 2.4 用户交互模块（均需登录）

#### 9. 点赞 / 取消点赞
```
POST /api/recipes/:id/like
```
**请求头：** `Authorization: Bearer <token>`

**说明：** 切换点赞状态，返回当前菜谱最新点赞数。

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "liked": true,
    "likes": 1255
  }
}
```

#### 10. 收藏 / 取消收藏
```
POST /api/recipes/:id/favorite
```
**请求头：** `Authorization: Bearer <token>`

**说明：** 切换收藏状态。

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "favorited": true
  }
}
```

#### 11. 获取我的收藏列表
```
GET /api/users/me/favorites?page=1&pageSize=10
```
**请求头：** `Authorization: Bearer <token>`

**说明：** 返回当前用户收藏的菜谱列表。

#### 12. 获取我的评价列表
```
GET /api/users/me/comments?page=1&pageSize=10
```
**请求头：** `Authorization: Bearer <token>`

**说明：** 返回当前用户发表的所有评论，附带关联菜谱信息。

#### 13. 获取我的点赞列表
```
GET /api/users/me/likes
```
**请求头：** `Authorization: Bearer <token>`

**说明：** 返回当前用户点赞过的菜谱 ID 数组，用于前端判断点赞状态。

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "recipeIds": ["r1", "r3", "r5"]
  }
}
```

### 2.5 用户认证模块

#### 14. 用户注册
```
POST /api/auth/register
```
**请求体：**
```json
{
  "username": "user1",
  "password": "123456"
}
```

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "token": "jwt_token",
    "user": {
      "id": "u1",
      "username": "user1",
      "avatar": "默认头像地址"
    }
  }
}
```

#### 15. 用户登录
```
POST /api/auth/login
```
**请求体：**
```json
{
  "username": "user1",
  "password": "123456"
}
```

**响应示例：** 同注册，返回 token 和用户信息。

#### 16. 获取当前用户信息
```
GET /api/users/me
```
**请求头：** `Authorization: Bearer <token>`

**响应示例：**
```json
{
  "code": 200,
  "data": {
    "id": "u1",
    "username": "user1",
    "avatar": "..."
  }
}
```

#### 17. 更新用户信息
```
PUT /api/users/me
```
**请求头：** `Authorization: Bearer <token>`

**请求体：**
```json
{
  "username": "新昵称",
  "avatar": "新头像URL"
}
```

## 三、数据库表结构设计

### 3.1 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | VARCHAR(32) | PK | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名/昵称 |
| password | VARCHAR(255) | NOT NULL | 加密后的密码 |
| avatar | VARCHAR(500) | DEFAULT | 头像图片 URL |
| created_at | DATETIME | DEFAULT NOW() | 注册时间 |
| updated_at | DATETIME | DEFAULT NOW() | 更新时间 |

```sql
CREATE TABLE users (
  id VARCHAR(32) PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password VARCHAR(255) NOT NULL,
  avatar VARCHAR(500) DEFAULT 'https://picsum.photos/seed/default/200',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

---

### 3.2 父分类表 (parent_categories)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | VARCHAR(32) | PK | 分类唯一标识 |
| name | VARCHAR(50) | NOT NULL | 分类名称 |
| sort_order | INT | DEFAULT 0 | 排序顺序 |

```sql
CREATE TABLE parent_categories (
  id VARCHAR(32) PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  sort_order INT DEFAULT 0
);
```

---

### 3.3 子分类表 (sub_categories)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | VARCHAR(32) | PK | 子分类唯一标识 |
| parent_id | VARCHAR(32) | NOT NULL, FK | 所属父分类 ID |
| name | VARCHAR(50) | NOT NULL | 子分类名称 |
| image | VARCHAR(500) | NOT NULL | 分类封面图 URL |
| sort_order | INT | DEFAULT 0 | 排序顺序 |

```sql
CREATE TABLE sub_categories (
  id VARCHAR(32) PRIMARY KEY,
  parent_id VARCHAR(32) NOT NULL,
  name VARCHAR(50) NOT NULL,
  image VARCHAR(500) NOT NULL,
  sort_order INT DEFAULT 0,
  FOREIGN KEY (parent_id) REFERENCES parent_categories(id)
);
```

---

### 3.4 菜谱表 (recipes)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | VARCHAR(32) | PK | 菜谱唯一标识 |
| title | VARCHAR(100) | NOT NULL | 菜谱标题 |
| description | TEXT | NOT NULL | 菜谱描述 |
| image | VARCHAR(500) | NOT NULL | 封面图 URL |
| category_id | VARCHAR(32) | NOT NULL, FK | 所属子分类 ID |
| likes | INT | DEFAULT 0 | 点赞数 |
| difficulty | VARCHAR(10) | NOT NULL | 难度：简单/中等/困难 |
| time | VARCHAR(20) | NOT NULL | 制作时间，如"60分钟" |
| servings | INT | DEFAULT 1 | 几人份 |
| created_at | DATETIME | DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | DEFAULT NOW() | 更新时间 |

```sql
CREATE TABLE recipes (
  id VARCHAR(32) PRIMARY KEY,
  title VARCHAR(100) NOT NULL,
  description TEXT NOT NULL,
  image VARCHAR(500) NOT NULL,
  category_id VARCHAR(32) NOT NULL,
  likes INT DEFAULT 0,
  difficulty VARCHAR(10) NOT NULL CHECK (difficulty IN ('简单', '中等', '困难')),
  time VARCHAR(20) NOT NULL,
  servings INT DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES sub_categories(id)
);
```

---

### 3.5 菜谱食材表 (recipe_ingredients)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | INT | PK, AUTO_INCREMENT | 自增 ID |
| recipe_id | VARCHAR(32) | NOT NULL, FK | 所属菜谱 ID |
| name | VARCHAR(50) | NOT NULL | 食材名称 |
| amount | VARCHAR(50) | NOT NULL | 用量 |
| sort_order | INT | DEFAULT 0 | 排序顺序 |

```sql
CREATE TABLE recipe_ingredients (
  id INT PRIMARY KEY AUTO_INCREMENT,
  recipe_id VARCHAR(32) NOT NULL,
  name VARCHAR(50) NOT NULL,
  amount VARCHAR(50) NOT NULL,
  sort_order INT DEFAULT 0,
  FOREIGN KEY (recipe_id) REFERENCES recipes(id) ON DELETE CASCADE
);
```

---

### 3.6 菜谱步骤表 (recipe_steps)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | INT | PK, AUTO_INCREMENT | 自增 ID |
| recipe_id | VARCHAR(32) | NOT NULL, FK | 所属菜谱 ID |
| step_number | INT | NOT NULL | 步骤序号 |
| description | TEXT | NOT NULL | 步骤描述 |
| image | VARCHAR(500) | DEFAULT NULL | 步骤配图 URL |
| ingredients_used | JSON | DEFAULT NULL | 该步骤使用的食材名称数组 |

```sql
CREATE TABLE recipe_steps (
  id INT PRIMARY KEY AUTO_INCREMENT,
  recipe_id VARCHAR(32) NOT NULL,
  step_number INT NOT NULL,
  description TEXT NOT NULL,
  image VARCHAR(500) DEFAULT NULL,
  ingredients_used JSON DEFAULT NULL,
  FOREIGN KEY (recipe_id) REFERENCES recipes(id) ON DELETE CASCADE
);
```

---

### 3.7 评论表 (comments)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | VARCHAR(32) | PK | 评论唯一标识 |
| recipe_id | VARCHAR(32) | NOT NULL, FK | 所属菜谱 ID |
| user_id | VARCHAR(32) | NOT NULL, FK | 评论用户 ID |
| text | TEXT | NOT NULL | 评论内容 |
| images | JSON | DEFAULT NULL | 评论图片 URL 数组 |
| created_at | DATETIME | DEFAULT NOW() | 评论时间 |

```sql
CREATE TABLE comments (
  id VARCHAR(32) PRIMARY KEY,
  recipe_id VARCHAR(32) NOT NULL,
  user_id VARCHAR(32) NOT NULL,
  text TEXT NOT NULL,
  images JSON DEFAULT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (recipe_id) REFERENCES recipes(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

---

### 3.8 用户点赞表 (user_likes)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | INT | PK, AUTO_INCREMENT | 自增 ID |
| user_id | VARCHAR(32) | NOT NULL, FK | 用户 ID |
| recipe_id | VARCHAR(32) | NOT NULL, FK | 菜谱 ID |
| created_at | DATETIME | DEFAULT NOW() | 点赞时间 |

```sql
CREATE TABLE user_likes (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id VARCHAR(32) NOT NULL,
  recipe_id VARCHAR(32) NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (recipe_id) REFERENCES recipes(id) ON DELETE CASCADE,
  UNIQUE KEY uk_user_recipe_like (user_id, recipe_id)
);
```

---

### 3.9 用户收藏表 (user_favorites)

| 字段名 | 类型 | 约束 | 说明 |
|:---|:---|:---|:---|
| id | INT | PK, AUTO_INCREMENT | 自增 ID |
| user_id | VARCHAR(32) | NOT NULL, FK | 用户 ID |
| recipe_id | VARCHAR(32) | NOT NULL, FK | 菜谱 ID |
| created_at | DATETIME | DEFAULT NOW() | 收藏时间 |

```sql
CREATE TABLE user_favorites (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id VARCHAR(32) NOT NULL,
  recipe_id VARCHAR(32) NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (recipe_id) REFERENCES recipes(id) ON DELETE CASCADE,
  UNIQUE KEY uk_user_recipe_fav (user_id, recipe_id)
);
```

## 四、表关系图

```
parent_categories (1)
       |
       | (1:N)
       v
sub_categories (N)  <---  category_id  ---  recipes (N)
                                                   |
          ┌────────────────────────────────────────┤
          │                                        │
          │ (1:N)                                  │ (1:N)
          v                                        v
recipe_ingredients                      recipe_steps
          │                                        │
          │ (1:N)                                  │ (1:N)
          v                                        v
     ingredients_used (JSON)                     images (VARCHAR)

users (1)
  |
  |---(1:N)---> comments
  |---(1:N)---> user_likes  <---(N:M)---> recipes
  |---(1:N)---> user_favorites <---(N:M)---> recipes
```

## 五、前端调用接口对应关系
